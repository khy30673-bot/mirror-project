# 오늘의 거울 🪞

따뜻한 메이크업 거울 웹 페이지. 카메라로 자신을 비추고, 노브를 돌리면 오르골 음악과 함께 불빛 색이 바뀌어요.

## 파일 구조

```
mirror-project/
└── index.html   ← 이거 하나가 전부예요
```

HTML, CSS, JavaScript가 한 파일에 다 들어있어요. 입문자가 전체 흐름을 한눈에 보기 좋도록.

## 로컬에서 먼저 테스트하기

**중요**: 그냥 `index.html`을 더블클릭해서 열면 카메라가 안 켜져요.
브라우저는 `file://` 경로에서는 카메라 접근을 막거든요. `https://` 또는 `localhost`여야 해요.

가장 쉬운 방법은 VS Code의 **Live Server** 확장 프로그램을 쓰는 거예요:

1. VS Code에서 이 폴더 열기
2. 확장 프로그램에서 "Live Server" 설치
3. `index.html` 우클릭 → "Open with Live Server"
4. 브라우저에 자동으로 열리고, 카메라 권한 요청 팝업이 뜸 → 허용

## Vercel 배포 (3가지 방법, 위에서부터 쉬운 순)

### 방법 A: 가장 쉬움 — 폴더 드래그앤드롭

1. [vercel.com](https://vercel.com) 가입 (GitHub 계정으로 로그인하면 편함)
2. 대시보드에서 **"Add New..." → "Project"** 클릭
3. 페이지를 살짝 내려보면 **"deploy a template"** 옆쯤에 직접 업로드 옵션이 있을 거예요.
   (UI는 자주 바뀌니까 안 보이면 방법 B로 가세요)
4. `mirror-project` 폴더를 드래그앤드롭
5. **Deploy** 클릭
6. 30초쯤 뒤에 `xxx.vercel.app` 같은 URL이 생성됨

### 방법 B: GitHub 연결 (한 번 세팅하면 가장 편함)

1. GitHub에 새 저장소(repository) 만들기 (예: `mirror-project`)
2. `index.html`을 그 저장소에 올리기
   - GitHub 웹사이트에서 "Add file" → "Upload files"로 그냥 드래그해도 됨
3. Vercel 대시보드 → **"Add New..." → "Project"**
4. GitHub 저장소 선택 → **Import**
5. 설정은 다 그대로 두고 **Deploy** 클릭
6. 끝. 앞으로 GitHub에 코드를 푸시할 때마다 자동으로 재배포됨

### 방법 C: 명령어로 (Vercel CLI)

Node.js가 설치되어 있어야 해요.

```bash
npm install -g vercel
cd mirror-project
vercel
```

질문이 몇 개 나오는데 다 엔터(기본값)로 넘기면 됨.

## 작동 원리 (입문자용 설명)

코드 전체에 주석을 달아뒀지만, 큰 흐름만 다시 정리하면:

### 1. 카메라 켜기

```javascript
navigator.mediaDevices.getUserMedia({ video: true })
```

이 한 줄이 핵심이에요. 브라우저가 사용자에게 "카메라 써도 돼요?" 권한을 요청하고, 허락하면 `<video>` 태그에 영상을 연결.

### 2. 거울 효과

```css
transform: scaleX(-1);
```

카메라 영상을 가로로 뒤집어주면 진짜 거울처럼 좌우가 반전돼요. 진짜 거울이 빛을 반사해서 좌우가 뒤집혀 보이는 것과 같은 원리.

### 3. 상하 반전

```css
transform: scaleX(-1) scaleY(-1);
```

`scaleY(-1)`을 추가하면 세로도 뒤집힘. 버튼 클릭하면 JavaScript가 이 클래스를 토글(껐다 켰다)함.

### 4. 노브 회전

마우스가 움직일 때마다:
1. 노브 중심에서 마우스까지의 각도를 계산
2. 직전 각도와 비교해서 얼마나 움직였는지(delta) 구함
3. delta가 양수(시계방향)일 때만 누적
4. 일정량 누적되면 색깔 단계를 다음으로 넘김

### 5. 오르골 음악

Web Audio API로 사인파(부드러운 톤) + 삼각파(약간의 금속성)를 합쳐서 오르골 음색을 흉내냄. 빠르게 시작했다가 천천히 사라지는 음 envelope이 오르골 특유의 "띠잉~" 느낌을 만들어요.

## 알아두면 좋은 점

- **카메라 권한**: 브라우저는 도메인마다 권한을 따로 기억해요. 한 번 거부하면 주소창 자물쇠 아이콘에서 직접 풀어줘야 함.
- **모바일에서도 동작**: 터치 이벤트도 처리해뒀어요. 다만 iOS Safari는 첫 사용자 상호작용 후에만 오디오 재생을 허용하니, 노브를 돌리는 동작이 그 트리거 역할을 해요.
- **Vercel은 HTTPS 자동 제공**: 별도 설정 없이 카메라가 동작해요.
